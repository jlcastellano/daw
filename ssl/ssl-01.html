Certificados Digitales con OpenSSL

Introducción a la Criptografía Asimétrica

La criptografía asimétrica, también conocida como criptografía de clave pública, es un sistema criptográfico que utiliza un par de claves matemáticamente relacionadas:

Clave Privada
Se mantiene en secreto y nunca debe compartirse. Se utiliza para descifrar mensajes y firmar digitalmente documentos.
Clave Pública
Puede distribuirse libremente. Se utiliza para cifrar mensajes destinados al propietario de la clave privada y para verificar firmas digitales.

Principios Fundamentales

El sistema funciona mediante funciones matemáticas unidireccionales. Lo que se cifra con la clave pública solo puede descifrarse con la clave privada correspondiente, y viceversa. Esta propiedad permite dos funcionalidades principales:

Confidencialidad
Si Alice quiere enviar un mensaje secreto a Bob, cifra el mensaje con la clave pública de Bob. Solo Bob, con su clave privada, puede descifrar el mensaje.
Autenticación y No Repudio
Si Bob firma un documento con su clave privada, cualquiera con su clave pública puede verificar que realmente fue Bob quien lo firmó, y Bob no puede negar haberlo hecho.

Algoritmos Comunes

Los algoritmos más utilizados son RSA (Rivest-Shamir-Adleman), que se basa en la dificultad de factorizar números primos grandes, y ECC (Elliptic Curve Cryptography), que ofrece seguridad equivalente con claves más cortas. En este manual nos enfocaremos principalmente en RSA por su amplia adopción.

Certificados Digitales

Un certificado digital es un documento electrónico que vincula una clave pública con una identidad. Contiene información como el nombre del propietario, la clave pública, el emisor del certificado, el período de validez y una firma digital de la Autoridad Certificadora (CA) que lo emitió. Los certificados siguen el estándar X.509 y permiten establecer confianza en entornos donde las partes no se conocen previamente.

1. Preparación del Entorno

1.1 Crear Estructura de Directorios

Primero crearemos una estructura organizada para nuestra infraestructura de CA:

# Crear directorios para la CA raíz
mkdir -p ca/root-ca/{private,certs,newcerts,crl,csr}
chmod 700 ca/root-ca/private

# Crear directorios para la CA intermedia
mkdir -p ca/intermediate-ca/{private,certs,newcerts,crl,csr}
chmod 700 ca/intermediate-ca/private

# Crear directorio para certificados de servidor y cliente
mkdir -p ca/{server,client}/{private,certs,csr}
chmod 700 ca/server/private ca/client/private

Explicación de los directorios:

  • private: Almacena las claves privadas (permisos restringidos)
  • certs: Almacena los certificados públicos
  • newcerts: Base de datos de certificados emitidos
  • crl: Listas de revocación de certificados
  • csr: Solicitudes de firma de certificados

1.2 Inicializar Archivos de Base de Datos

# Para la CA raíz
cd ca/root-ca
touch index.txt
echo 1000 > serial
echo 1000 > crlnumber

# Para la CA intermedia
cd ../intermediate-ca
touch index.txt
echo 1000 > serial
echo 1000 > crlnumber
cd ../..

Explicación:

  • index.txt: Base de datos de certificados emitidos
  • serial: Número de serie para el próximo certificado
  • crlnumber: Número de serie para la próxima CRL
