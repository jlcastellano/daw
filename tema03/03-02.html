Servidores Web

1.1. Aspectos generales de arquitecturas web

La arquitectura World Wide Web (WWW) de internet nos proporciona un modelo de programación extremadamente poderoso y flexible. Funciona de la siguiente manera: las aplicaciones y contenidos se presentan en formatos de datos estandarizados, y son localizados y mostrados por aplicaciones cliente (los navegadores web), que solicitan recursos específicos a un servidor. El servidor responde enviando los datos codificados según formatos estándar que el navegador puede interpretar y mostrar.

Los estándares web definen muchos de los mecanismos necesarios para construir un entorno de aplicaciones de propósito general. Estos incluyen:

  • Modelo estándar de nombres: todos los servidores y contenidos de la web se identifican mediante un Localizador Uniforme de Recursos (URL). Esto permite localizar cualquier recurso en internet de manera única e inequívoca.
  • Tipificación de contenido: a todos los contenidos en la web se les asigna un tipo específico (por ejemplo, imagen JPEG, documento PDF, vídeo MP4), lo que permite que los navegadores los interpreten y muestren correctamente.
  • Formatos de contenido estándar: todos los navegadores modernos soportan un conjunto común de formatos y tecnologías estándar. Entre los más importantes encontramos HTML (el lenguaje que estructura las páginas web), JavaScript (que añade interactividad), CSS (que define la apariencia visual), y formatos multimedia como MP4 para vídeo o JPEG y PNG para imágenes.
  • Protocolos estándar: estos protocolos garantizan que cualquier navegador pueda comunicarse con cualquier servidor web, independientemente del fabricante o sistema operativo. El protocolo más utilizado en la web es HTTP (Protocolo de Transferencia de Hipertexto), que opera sobre la arquitectura de protocolos TCP/IP (Protocolo de Control de Transmisión/Protocolo de Internet), que es la base de las comunicaciones en internet.

Esta infraestructura estandarizada permite a los usuarios acceder a una cantidad prácticamente ilimitada de aplicaciones y servicios desarrollados por terceros. Al mismo tiempo, facilita a los desarrolladores crear aplicaciones y servicios que pueden ser utilizados por una comunidad global de usuarios, sin preocuparse por incompatibilidades técnicas.

Los aspectos clave que debemos considerar al diseñar una arquitectura web son:

  • Escalabilidad: la capacidad del sistema para crecer y manejar un mayor número de usuarios o mayor volumen de datos sin perder rendimiento.
  • Separación de responsabilidades: dividir la aplicación en capas o módulos independientes, donde cada uno tiene una función específica y bien definida.
  • Portabilidad: la capacidad de que la aplicación funcione en diferentes plataformas y dispositivos sin requerir modificaciones significativas.
  • Utilización de componentes en los servicios de infraestructura: usar módulos reutilizables que simplifiquen el desarrollo y mantenimiento.
  • Gestión de las sesiones del usuario: mantener el contexto y estado de cada usuario mientras navega por la aplicación.
  • Aplicación de patrones de diseño: utilizar soluciones probadas y estandarizadas para problemas comunes de desarrollo.

El funcionamiento de los servicios web modernos requiere de tres elementos fundamentales:

  • Proveedor del servicio web: es quien diseña, desarrolla e implementa el servicio, y lo pone a disposición de otros, ya sea dentro de la misma organización o públicamente en internet.
  • Consumidor del servicio: es quien accede al servicio para utilizar las funcionalidades que este ofrece. Puede ser un usuario final a través de un navegador, o bien otra aplicación.
  • Agente del servicio: actúa como intermediario entre proveedor y consumidor, facilitando la publicación, búsqueda y localización de servicios disponibles.
