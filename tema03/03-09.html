Servidores Web

2.3. Tipos de aplicaciones web

Las aplicaciones web abarcan un espectro muy amplio de funcionalidades y complejidad. Desde simples páginas informativas hasta complejas plataformas de comercio electrónico, reservas, redes sociales, o herramientas de productividad. Establecer una clasificación precisa es complejo debido a la enorme variedad existente, pero podemos organizarlas según diferentes criterios:

Según su naturaleza de contenido y tecnología

Sitios web estáticos

  • Contenido fijo que no cambia sin intervención manual.
  • Implementados en HTML, CSS, y posiblemente JavaScript básico.
  • Pueden incluir elementos dinámicos visuales (animaciones CSS, JavaScript) pero el contenido en sí no cambia.
  • Ejemplos: páginas de presentación de pequeñas empresas, portfolios personales, sitios informativos básicos.
  • Ventajas: muy rápidos, económicos de hospedar, seguros (menos vectores de ataque).
  • Desventajas: difíciles de actualizar sin conocimientos técnicos.
  • En 2025, los generadores de sitios estáticos (Gatsby, Next.js en modo estático, Hugo) han revitalizado este enfoque, combinando las ventajas de lo estático con herramientas modernas de desarrollo.

Sitios web dinámicos

  • Contenido generado en tiempo real según la petición del usuario.
  • Interactúan con bases de datos para recuperar y presentar información.
  • Implementados con tecnologías del servidor (PHP, ASP.NET, Node.js, etc.).
  • Permiten personalización según el usuario.
  • Ejemplos: blogs, sitios de noticias, buscadores, plataformas educativas.
  • Se actualizan fácilmente mediante paneles de administración.

Aplicaciones web de una sola página (SPA)

  • Cargan una única página HTML y actualizan dinámicamente el contenido sin recargar.
  • Implementadas con frameworks JavaScript (React, Vue, Angular).
  • Experiencia de usuario fluida, similar a aplicaciones nativas.
  • Se comunican con el servidor mediante APIs (generalmente REST o GraphQL).
  • Ejemplos: Gmail, Google Maps, Facebook, Twitter/X.
  • Requieren JavaScript habilitado en el navegador.

Aplicaciones web progresivas (PWA)

  • Combinan características de web y aplicaciones móviles nativas.
  • Funcionan offline, se pueden instalar, envían notificaciones.
  • Responsivas y adaptables a cualquier dispositivo.
  • Ejemplos: Twitter Lite, Starbucks, Uber.
  • En 2025, cada vez más empresas optan por PWAs en lugar de aplicaciones nativas.

Según su funcionalidad

Portales web

  • Puntos de entrada que agregan múltiples servicios y secciones.
  • Ofrecen: noticias, foros, correo electrónico, buscadores, servicios personalizados.
  • Ejemplos: Yahoo, MSN (aunque menos relevantes en 2025), intranets corporativas.

Tiendas online (eCommerce)

  • Plataformas de comercio electrónico.
  • Funcionalidades: catálogo de productos, carrito de compra, pasarela de pago, gestión de pedidos, sistema de valoraciones.
  • Plataformas populares en 2025: Shopify, WooCommerce, Magento, PrestaShop.
  • Integración con sistemas de pago, logística, inventario.
  • Ejemplos: Amazon, eBay, tiendas online de marcas.

Redes sociales

  • Plataformas para conexión e interacción entre usuarios.
  • Características: perfiles de usuario, publicación de contenido, mensajería, feeds personalizados.
  • Ejemplos: Facebook, Instagram, LinkedIn, TikTok, X (Twitter).
  • Algorítmicamente complejas para manejar millones de usuarios simultáneos.

Sistemas de gestión de contenidos (CMS)

  • Plataformas que permiten crear y gestionar contenido sin conocimientos técnicos.
  • Interfaz de administración intuitiva.
  • Plataformas populares: WordPress (el CMS más usado mundialmente), Drupal, Joomla; en 2025 también headless CMS como Strapi, Contentful.
  • Permiten a no-técnicos actualizar sitios web fácilmente.

Aplicaciones empresariales (ERP, CRM)

  • Sistemas de gestión empresarial accesibles vía web.
  • ERP (Enterprise Resource Planning): gestión integral de la empresa.
  • CRM (Customer Relationship Management): gestión de relaciones con clientes.
  • Ejemplos: Salesforce, SAP, Odoo.
  • Aplicaciones complejas con múltiples módulos integrados.

Plataformas educativas (LMS)

  • Sistemas de gestión del aprendizaje.
  • Funcionalidades: cursos online, evaluaciones, seguimiento del progreso, foros, contenido multimedia.
  • Ejemplos: Moodle, Canvas, Google Classroom, Coursera, Udemy.
  • Muy importantes especialmente desde 2020-2025 con el auge del aprendizaje online.

Aplicaciones de colaboración y productividad

  • Herramientas para trabajo colaborativo.
  • Ejemplos: Google Workspace (Docs, Sheets, Drive), Microsoft 365, Notion, Trello, Slack.
  • Permiten edición simultánea, comunicación en tiempo real, gestión de proyectos.

Servicios de streaming

  • Plataformas de contenido multimedia on-demand.
  • Ejemplos: Netflix, YouTube, Spotify, Disney+, Twitch.
  • Requieren infraestructura robusta para distribución de contenido.
  • Adaptación de calidad según ancho de banda (streaming adaptativo).

Aplicaciones de reservas y booking

  • Sistemas para reservar servicios o recursos.
  • Ejemplos: Booking.com, Airbnb, sistemas de reserva de citas médicas, reservas de restaurantes.
  • Integración con calendarios, pagos, confirmaciones automáticas.
