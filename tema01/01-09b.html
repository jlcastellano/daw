Estructura Detallada por Niveles

Nivel 1: Dominio Base

dc=ejemplo,dc=com

Es la raíz del árbol LDAP. Representa el dominio de la organización.

Nivel 2: Unidad Organizativa Principal

ou=Departamentos,dc=ejemplo,dc=com

Contiene todos los departamentos de la empresa. Es el contenedor principal para la estructura organizativa.

Nivel 3: Departamentos Individuales

Se crean tres departamentos:

Explicación de los ObjectClass Utilizados

organizationalUnit (top)

Define unidades organizativas (contenedores). Se usa para crear la estructura de departamentos y contenedores de usuarios.

inetOrgPerson

Proporciona información personal y de contacto de los usuarios. Incluye atributos como cn (nombre completo), sn (apellido), givenName (nombre) y mail (correo electrónico).

posixAccount

Proporciona información de cuenta Unix/Linux. Incluye atributos como uid, uidNumber, gidNumber, homeDirectory y loginShell. Permite la autenticación de usuarios en sistemas Unix.

shadowAccount

Proporciona información de contraseña y políticas de expiración (Unix). Incluye el atributo userPassword que almacena la contraseña encriptada.

Esta estructura es típica para empresas pequeñas que necesitan organizar usuarios por departamentos con autenticación Unix/Linux integrada. La jerarquía permite una gestión centralizada de usuarios y facilita la aplicación de políticas de acceso por departamento.
